# NanoCare AI MVP Plan

## Week 1

- Frontend command center layout
- Sector login for Administrativo, Enfermaria and Cirurgias
- Internal pages for command, patients, beds, alerts and AI analytics
- Hospital Live Map
- Login shell
- Cyber hospital visual system

## Week 2

- Go hospital core service
- Patient API
- Bed and sector status endpoints

## Week 3

- Python FastAPI triage service
- AI risk scoring prototype
- Predictive metrics for priority, average wait time and fatigue

## Week 4

- WebSocket real-time updates
- Alert engine
- Voice alert integration
- Docker Compose local environment

## Current Phase 2 Additions

- Realtime simulation hook in the frontend
- Sector-specific login and initial view routing
- Patient monitoring page
- Bed and sector operations page
- Alert history page
- AI analytics and fatigue heatmap page
- Backend service skeletons for Node, Go, Python and WebSocket

## Phase 3 Recruiter Demo Mode

- Operator login context with permissions, shift and last access
- Demo Mode badge for guided portfolio presentation
- AI action recommendations with confidence and expected impact
- Manual triage simulator with risk explanation
- Live event feed for realtime operational updates
- SLA and recommendation block inside alerts
- System Architecture screen explaining frontend, gateway, Go core, Python AI, Redis/WebSocket and PostgreSQL
- Gateway endpoints for patients, sectors, alerts and triage proxy

## Phase 4 Above-Average Demo

- AI body scanner with rotating patient model and scan beam
- Smart patient record with history, timeline and generated clinical summary
- Presentation Mode overlay for guided recruiter storytelling
- Executive report flow using browser print/PDF
- Public Case page with problem, solution, stack and expected impact
- Frontend WebSocket connection with fallback to local realtime simulation
